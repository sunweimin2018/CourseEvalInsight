# 学生成绩质量检测报告生成系统功能设计文档

# `1\. 文档概述`

## `1\.1 文档目的`

`本文档为**学生成绩质量检测报告自动生成功能**的详细设计文档，用于指导前后端开发、AI代码生成（Claude Code）落地实现。文档明确了系统的输入文件规范、数据解析逻辑、业务流程、功能模块、数据分析规则、可视化图表生成、报告输出格式，统一开发标准，保障功能可落地、可复用、可扩展。`

## `1\.2 适用范围`

`适用于前后端研发人员、AI代码生成工具（Claude Code）、测试人员，覆盖文件上传、数据解析、信息填充、成绩数据分析、图表生成、报告合成与导出全流程功能开发。`

## `1\.3 输入文件说明`

`系统核心输入三类固定格式文件，由用户前端上传，为报告生成唯一数据源：`

1. `**课程大纲（Word文档，\.docx）**：提供课程基础信息、考核标准、教学目标等文本类信息，用于填充报告基础说明模块；`

2. `**学生基本信息表（Excel，\.xlsx/\.xls）**：提供学生班级、学号、姓名、性别、生源等基础档案信息，用于报告学生概况模块；`

3. `**学生成绩表（Excel，\.xlsx/\.xls）**：提供学生各科成绩、总分、排名等核心数据，用于数据分析、统计计算、图表生成。`

## `1\.4 输出成果`

`最终生成完整的**学生成绩质量检测报告**，包含文本综述、数据统计表格、可视化分析图表，支持在线预览、Word/PDF导出。`

# `2\. 整体需求分析`

## `2\.1 功能需求`

1. `多文件批量上传、格式校验、文件解析与数据存储；`

2. `课程大纲文本信息智能提取，自动填充报告课程概述、考核依据模块；`

3. `学生基本信息解析、汇总统计，生成学生整体概况内容；`

4. `学生成绩数据清洗、合规校验、多维度数据分析；`

5. `自动生成成绩统计数据表、可视化分析图表（柱状图、折线图、饼图、分布图等）；`

6. `文本内容、数据表格、图表自动合成完整检测报告；`

7. `报告在线预览、编辑、导出（Word/PDF）；`

8. `数据缓存、异常提示、操作日志记录。`

## `2\.2 非功能需求`

1. `**兼容性**：支持主流docx、xlsx格式文件，兼容Windows、Mac端上传文件；`

2. `**准确性**：数据解析准确率100%，成绩统计、计算逻辑无误差；`

3. `**高效性**：单批次1000条以内学生数据，报告生成耗时≤10s；`

4. `**稳定性**：支持异常文件拦截、数据缺失容错、重复上传覆盖机制；`

5. `**可扩展性**：后续可新增分析维度、图表类型、报告模板，无需重构核心逻辑。`

# `3\. 系统整体架构设计`

`采用**前后端分离架构**，分为前端交互层、业务服务层、数据解析层、存储层，Claude Code可按分层逻辑模块化生成代码，低耦合、高复用。`

## `3\.1 架构分层`

### `3\.1\.1 前端交互层`

`负责用户操作交互，核心功能：多文件上传、上传进度展示、文件格式校验、报告预览、报告编辑、导出下载、数据分析结果可视化展示。`

### `3\.1\.2 业务服务层（后端核心）`

`承接前端请求，统筹核心业务逻辑：文件接收、调用解析接口、数据整合、数据分析、图表数据生成、报告模板渲染、文件导出。`

### `3\.1\.3 数据解析层`

`专属文件解析模块，独立处理三类文件解析逻辑：Word大纲解析、Excel学生信息解析、Excel成绩数据解析，完成数据清洗、去重、容错处理。`

### `3\.1\.4 存储层`

`临时存储上传文件、解析后结构化数据、生成的报告文件，支持会话缓存，任务结束后可自动清理冗余数据。`

# `4\. 核心功能模块详细设计`

`系统拆分为6大核心模块，各模块独立运作、数据联动，Claude Code可分模块迭代开发。`

## `4\.1 文件上传与校验模块`

### `4\.1\.1 上传规则`

1. `支持批量上传，单次必须同时/分批次上传**课程大纲、学生基本信息表、学生成绩表**三类文件；`

2. `格式限制：课程大纲仅支持\.docx，信息表、成绩表仅支持\.xlsx/\.xls；`

3. `大小限制：单文件≤20MB，禁止空文件、损坏文件上传；`

4. `重复上传：同名文件自动覆盖，保留最新上传数据。`

### `4\.1\.2 校验逻辑`

1. `文件后缀、MIME类型校验，拦截非法格式；`

2. `文件完整性校验，拦截损坏、无法打开的文件；`

3. `核心字段校验：学生信息表、成绩表必须包含关键字段，缺失则提示上传文件格式错误。`

## `4\.2 文件数据解析模块`

### `4\.2\.1 课程大纲（Word）解析逻辑`

`采用富文本解析规则，提取固定核心信息，用于填充报告**课程基本信息表、考核评价依据、教学目标**模块：`

1. `提取字段：课程名称、课程编号、授课班级、开课院系、课程性质、课程类型、修课人数、课序号、总学时、学分、授课教师、选用教材、课程目标、课程考核及成绩评定。`

   `1.1 选用教材字段的值为标题“教材及参考资料”下的“课程教材”下的内容。不包含标题“**参考教材及网站**”及其后的内容。`

   `1.2 课程目标字段的值为标题“课程目标”下的内容，不包含该标题下表格及表格上面的表格说明。`

   `1.3 课程考核及成绩评定字段的值来自于标题课程考核及成绩评定到标题教材及参考资料以上的所有内容`

2. `解析规则：匹配文档标题、一级/二级标题对应段落内容，自动过滤空段落，保留原内容的设定，以方便最终写入到质量检测报告中去;`

3. `容错处理：缺失部分字段则报告对应位置显示“未填写”，不中断流程。`

### `4\.2\.2 学生基本信息表（Excel）解析逻辑`

`结构化解析学生档案数据，汇总学生整体情况，用于报告**被测学生概况**模块。`

`提取字段：学号、姓名、班级、性别、总人数；`

`统计字段：总人数，男生人数、女生人数，所含班级。`

`解析后统计维度：班级总人数、男女比例、各班级人数分布。`

### `4\.2\.3 学生成绩表（Excel）解析逻辑`

`核心数据解析模块，为数据分析提供数据源。`

`默认必填字段：学生成绩表表头的各项字段；`

`数据清洗规则：`

1. `剔除空白行、无效数据行（无学号/无成绩）；`

2. `成绩空值标记为“缺考”，不参与平均分、合格率计算；`

3. `校验成绩数值合法性（0\-100分），超范围数据标记为异常数据并记录；`

4. `按学号去重，保留最新单条成绩数据。`

## `4\.3 报告文本内容生成模块`

`基于解析后的课程信息、学生基本信息表，自动生成段落文本。`

`内容包含：课程基本信息表、课程目标、课程评价标准、课程评价结果、课程持续改进方案及措施共五大模块。`
