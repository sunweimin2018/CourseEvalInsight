# 一、项目概述

## 1.1 项目背景

日常业务办公、数据统计场景中，用户普遍存在多份Excel零散数据需要汇总分析的痛点，传统人工汇总、统计、制表、写报告方式效率极低，且易出现数据统计错误、格式不统一、分析维度单一、报告复用性差等问题。目前市面工具多为单文件解析，无法实现多Excel文件数据关联融合、批量分析、动态生成标准化图文分析报告。基于此，开发一套基于Vue3\+Django的多Excel数据动态分析报告系统，支持批量上传多份Excel、自动清洗融合数据、多维度智能分析、可视化图表展示、一键生成PDF/Word数据分析报告，实现数据统计与报告产出全流程自动化。

## 1.2 项目目标

本次开发聚焦多Excel文件数据自动化处理与智能报告生成，实现零代码快速数据分析，替代人工统计撰写报告，具体目标如下：

- **业务目标**：实现多Excel零散数据自动汇总、关联分析，标准化输出专业数据分析报告，大幅降低数据统计与报告撰写人力成本，提升数据处理效率90%以上

- **功能目标**：支持多Excel批量上传、数据智能清洗、多文件数据融合、自定义分析维度、可视化图表渲染、动态报告生成与多格式导出、历史报告管理

- **性能目标**：单批次支持10份以内Excel文件批量解析，千级数据解析耗时≤3s，报告生成耗时≤5s，页面加载≤2s

- **体验目标**：极简操作流程，全程可视化操作，无代码操作，异常数据实时提示，报告实时预览、一键导出

## 1.3 项目范围

### 1.3.1 包含范围（本次必做功能）

1. 前端：登录页面、首页数据看板、多Excel批量上传页面、数据预览与清洗页面、分析维度配置页面、可视化图表展示页面、报告预览与导出页面、历史报告管理页面、个人中心页面；2. 后端：用户权限管理、Excel文件解析、数据清洗、多文件数据关联融合、多维度数据分析、图表数据计算、报告模板渲染、文件导出、数据存储与日志记录；3. 核心业务：支持.xlsx/.xls格式文件、自动去重/补全/异常数据过滤、自定义统计维度（求和/平均值/最大值/最小值/占比）、柱状图/折线图/饼图自动生成、标准化图文分析报告动态生成。

### 1.3.2 排除范围（本次不做功能）

1. 不支持CSV、WPS专属格式、加密Excel文件解析；2. 不支持超大万级以上海量数据高性能分片解析；3. 不支持自定义报告模板在线编辑；4. 不支持第三方云存储对接、多账号协作编辑；5. 不支持定时自动同步外部Excel数据。

## 1.4 目标用户

1. 普通业务用户：行政、运营、财务、市场等非技术办公人员，用于日常数据汇总、统计分析、工作报告生成；2. 管理员用户：系统管理员，负责用户管理、权限配置、数据日志查看、系统维护；3. 数据统计人员：高频处理多份表格数据、定期输出分析报告的岗位人员。

# 三、业务整体架构

## 3.1 业务架构流程

用户登录系统 → 批量上传多份Excel、docx文件 → 前端预处理校验 → 后端接收文件 → 批量解析文件数据 → 智能数据清洗（去重、补空、剔除异常）→ 多文件数据关联融合 → 用户自定义分析维度与统计规则 → 后台批量计算分析数据 → 生成可视化图表数据 → 动态渲染标准化分析报告 → 前端预览报告 → 导出Word/PDF报告 → 保存历史报告记录

## 3.2 模块依赖关系

用户权限模块（基础依赖）→ Excel文件处理模块（核心输入）→ 数据清洗融合模块（数据预处理）→ 数据分析计算模块（核心业务）→ 可视化图表模块（数据输出）→ 报告生成导出模块（最终产出）→ 历史记录模块（数据存储）

## 4.2 模块二：多Excel文件上传与解析模块（核心）

### 4.2.1 模块概述

支持用户按课程/班级/学期维度上传三类固定格式文件，为报告生成唯一数据源。前端完成课程/班级/学期选择与管理、文件分类上传、基础格式大小校验，后端按维度归类存储、批量解析文件数据，实现多文件数据读取、字段匹配、原始数据预览，为后续数据清洗与分析提供数据源支撑。课程名称、班级、学期为用户个人信息，不在各用户间共享。

**三类输入文件及作用：**

1. **课程大纲（Word文档，.docx）**：提供课程基础信息、考核标准、教学目标等文本类信息，用于填充报告基本信息表、课程目标、课程评价标准模块；
2. **学生基本信息表（Excel，.xlsx/.xls）**：提供学生班级、学号、姓名、性别、生源等基础档案信息，用于报告被测学生概况模块，自动统计总人数、男女人数、班级分布；
3. **学生成绩表（Excel，.xlsx/.xls）**：提供学生各科成绩、总分等核心数据，用于数据分析、统计计算、图表生成，最终写入报告课程评价结果模块。

**上传规则**：单次可分批上传三类文件；格式限制：课程大纲仅支持.docx；信息表、成绩表仅支持.xlsx/.xls；单文件大小限制：≤20MB；同名同类型文件自动覆盖，保留最新数据。

### 4.3.3 三类文件解析规则（AI强制实现）

**一、课程大纲（Word .docx）解析规则：**

采用标题匹配规则提取固定核心信息，用于填充报告课程基本信息表和文本模块：

| 提取字段 | 提取规则 |
|----------|----------|
| 课程名称 | 文档标题或“课程名称”对应内容 |
| 课程编号 | “课程编号”对应内容 |
| 授课班级 | “授课班级”或“教学对象”对应内容 |
| 开课院系 | “开课院系”对应内容 |
| 课程性质 | “课程性质”对应内容（必修/选修等） |
| 课程类型 | “课程类型”对应内容 |
| 修课人数 | “修课人数”或学生信息表统计总人数 |
| 课序号 | “课序号”对应内容 |
| 总学时/学分 | “总学时”“学分”对应内容 |
| 授课教师 | “授课教师”或“教师”对应内容 |
| 选用教材 | 标题“教材及参考资料”下“课程教材”内容，不含“参考教材及网站”及其后内容 |
| 课程目标 | 标题“课程目标”下文本内容，不含该标题下的表格及其说明 |
| 课程考核及成绩评定 | 标题“课程考核及成绩评定”至标题“教材及参考资料”之间的所有内容 |

解析要求：自动过滤空段落，保留原文格式以便写入报告；缺失字段在报告中显示”未填写”，不中断流程。

**Word大纲解析实现要点（AI开发强制遵守）：**

1. **标题匹配规则**：标题检测必须采用”去编号前缀后精确匹配”，不能使用子串包含匹配。反例：`”课程目标” in “课程目标1：系统掌握...”` 会误判为标题导致内容截断。正例：去除中文编号（一、二、三…）、数字编号（1. 2.）、括号编号（(一)（一）(1)）后，与目标标题完全相等才算匹配。

2. **表格字段提取双模式**：Word表格中标签与值的布局存在两种模式，必须同时支持：
   - **同单元格模式**：标签与值在同一单元格内，如 `”课程编号：MAN259”`，使用正则 `标签[：:\s]+(.+)` 提取
   - **相邻单元格模式**：标签在 cell[i]（如 `”课程名称：”`），值在 cell[i+1]（如 `”程序设计基础”`）。标签匹配需同时支持 ASCII 和全角冒号两种后缀，且支持 `endswith` 匹配以处理标签带前缀的情况（如 `”大纲执笔人：”` 匹配标签 `”执笔人”`）。相邻单元格的值若本身也是已知标签（如 `”总学时”` 在 `”学时：”` 右侧），须跳过避免误提取

3. **字段标签别名体系**：不同院校的课程大纲用词不统一，核心字段须配置多别名映射：
   - 开课院系：`开课院系`、`授课单位`、`开课单位`
   - 授课教师：`授课教师`、`执笔人`
   - 总学时：`总学时`、`学时数`、`学时`
   - 学分：`学分`

4. **节选内容提取**：课程目标、选用教材、课程考核三个大段按标题边界提取文本。选用教材只取”课程教材”至”参考教材”之间的内容，排除参考书目。

**二、学生基本信息表（Excel .xlsx/.xls）解析规则：**

提取字段：学号、姓名、班级、性别。自动统计维度：

- 总人数：去除空行后的有效学生记录数
- 男生人数 / 女生人数（按性别字段分组统计）
- 所含班级列表及各班人数分布

**学生信息表解析实现要点：**

1. **双层表头检测**：部分Excel文件的真实列名不在pandas解析出的headers中（显示为"Unnamed: N"），而是出现在第一行数据行中（子表头行）。解析时需检测首行是否包含"学号""姓名""性别""班级"等关键词，若包含则识别为子表头并据此建立列映射。

2. **列名空格容错**：真实Excel中列名可能含有多余空格（如 `"性  别"` 而非 `"性别"`），匹配时需先做去空格归一化（`str.replace(' ', '')`）。

3. **数据行跳过**：子表头行本身不计入统计数据，有效数据从子表头的下一行开始。

**三、学生成绩表（Excel .xlsx/.xls）解析与清洗规则：**

数据清洗规则：
- 剔除空白行、无效数据行（无学号或无成绩的记录）
- 成绩空值标记为“缺考”，不参与平均分、合格率计算
- 校验成绩数值合法性（0-100分），超范围数据标记为异常并记录
- 按学号去重，保留最新单条成绩数据
- 文本字段空值填充”无”，数值字段空值在补全时填充0

**成绩表解析实现要点：**

1. **数值列识别与排除规则**：识别成绩列时采用”≥80%非空值可解析为float”的启发式规则，但必须显式排除标识类列名（`学号`、`序号`、`编号`、`学生证号`、`ID`、`No`），避免将学号（如2024320519）误判为成绩列参与统计分析。排除名单采用精确列名匹配，不参与启发式计算。

2. **缺考标记识别**：除空值外，`缺考`、`免修`、`-`、`/` 等特殊标记均视为缺考，不计入有效成绩，不参与均值/标准差/及格率计算，但在总人数中保留计数（缺考人数统计）。

3. **分数段统计**：按 [90-100)、[80-89)、[70-79)、[60-69)、[0-59) 五档自动分组统计各分数段人数与占比，支持柱状图+饼图双图表渲染。

- 成绩核心指标汇总：有效成绩数、缺考人数、平均分、最高分、最低分、合格率、优秀率（由成绩表分析得出）

**报告生成与导出实现要点：**

1. **数据源优先级**：成绩数据优先使用用户编辑后保存的快照（WorkbookSnapshot），若无快照则回退到原始上传文件解析结果，确保报告使用最新编辑数据。

2. **报告记录唯一性**：同一（课程+班级+学期+用户）组合仅保留一份报告记录，再次生成时upsert更新而非重复创建，保证报告数据的时效性。

3. **Word生成**：使用python-docx构建报告，输出目标为BytesIO流（非临时文件），通过FileResponse直接返回流数据避免磁盘残留。报告状态流转：draft（草稿）→ completed（已生成）→ exported（已导出）。

4. **前端导出须携带认证Token**：导出接口返回二进制流（`Content-Type: application/vnd.openxmlformats-officedocument.wordprocessingml.document`），前端不能使用裸 `<a>` 标签下载（无法携带JWT Authorization头），必须通过Axios请求配置 `responseType: 'blob'`，手动携带token头，收到响应后使用 `URL.createObjectURL` + 动态 `<a>` 触发下载，完成后 `URL.revokeObjectURL` 释放。

5. **前端响应拦截器兼容**：Axios拦截器默认假设所有响应为JSON格式（`{code, msg, data}`）并自动解包 `data` 字段。二进制下载请求需绕过该拦截器直接发起原始axios请求，且调用方在读取JSON接口响应时直接使用解包后的数据，不再二次访问 `.data` 属性。
